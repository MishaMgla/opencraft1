# project map

navigation hub for AI agents. mirrors the source tree. each leaf doc describes one folder: purpose, key files, dependencies, sharp edges. **target ≤ 150 lines per leaf, hard cap ~250.**

> **scaffold status.** no source tree exists yet, so there are no leaf docs. as `src/` folders land, add one leaf doc per folder, register it in the `tree` and `task → doc` table below, and add a row to the pointer table in `AGENT_RULES.md`.

## how to read

1. read `AGENT_RULES.md` first — it has the rules and the pointer table.
2. find the leaf doc for your task in the table below.
3. load only that leaf. avoid loading the full tree.

## tree

```
docs/project-map/
  README.md              # this file
  glossary.md            # project-specific terms (empty until the domain emerges)
  index/
    README.md            # machine-readable index (routes / pages / components / exports)
    routes.json
    pages.json
    components.json
    exports.json
  server.md              # Go engine (cmd + internal packages)
  client.md              # web client (web/ modules)
```

## machine-readable index

`index/` holds four JSON files regenerated by the index generator (`scripts/build-index.ts`; alias as `yarn index` once a package manifest exists). consult them before greping for routes, pages, components, or exported symbols. for ground truth (signatures, types), still read the file. see `index/README.md`.
